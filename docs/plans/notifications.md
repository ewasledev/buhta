# Фича: уведомления об истечении подписок

## Мотивация

Администратор ведёт базу клиентов вручную. Без автоматических напоминаний легко пропустить момент, когда у клиента заканчивается подписка. Бот должен сам предупреждать о приближающихся истечениях — без необходимости открывать Telegram и что-то запрашивать вручную.

## Архитектура

```
AppModule
  └── ScheduleModule.forRoot()          # регистрирует планировщик (node-cron)
  └── NotificationsModule
        └── NotificationsService
              ├── @Cron('0 9 * * *')    # ежедневно в 09:00
              ├── SubscriptionsService  # запрос к БД
              ├── @InjectBot()          # прямой доступ к Telegraf
              └── ConfigService         # ADMIN_TELEGRAM_ID
```

**Почему `@nestjs/schedule`:** официальный NestJS-модуль поверх `node-cron`, интегрируется с DI-контейнером, не требует внешних сервисов (Redis, очереди). Для одной простой джобы — оптимальный выбор.

**Почему `@InjectBot()`:** проактивная отправка (не в ответ на апдейт от пользователя) требует прямого доступа к `bot.telegram.sendMessage()`. Декоратор `@InjectBot()` из `nestjs-telegraf` даёт инстанс `Telegraf`, зарегистрированный в `BotModule`.

## Схема работы джобы

```
09:00 каждый день
    │
    ├── findExpiringOn(today)    → список подписок, истекающих сегодня
    ├── findExpiringOn(tomorrow) → список подписок, истекающих завтра
    │
    ├── оба пусты? → ничего не делаем
    │
    └── есть данные → собираем текст → sendMessage(ADMIN_TELEGRAM_ID, text)
```

## Формат уведомления

```
⚠️ Подписка истекает СЕГОДНЯ:
• Иван Петров (до 25.04.2026)
• Мария Иванова (до 25.04.2026)

🔔 Подписка истекает ЗАВТРА:
• Алексей Сидоров (до 26.04.2026)
```

Если истекает только сегодня или только завтра — выводится только соответствующий блок.

## Настройка времени запуска

Время задаётся cron-выражением в `notifications.service.ts`:

```typescript
@Cron('0 9 * * *')  // 09:00 каждый день (время сервера)
```

Примеры:
- `'0 8 * * *'` — 08:00
- `'0 10 * * 1-5'` — 10:00 только по будням
- `'0 9,18 * * *'` — 09:00 и 18:00

Если сервер в UTC, а нужно московское время (UTC+3) — ставь на 3 часа раньше: `'0 6 * * *'`.

## Тестирование вручную

Чтобы проверить без ожидания 09:00 — временно заменить декоратор в `notifications.service.ts`:

```typescript
// вместо @Cron('0 9 * * *')
@Interval(5000)  // сработает через 5 секунд после старта
```

Создать тестовую подписку с `endDate = today` через бота, запустить `npm run start:dev`, убедиться что уведомление пришло. Затем вернуть `@Cron`.
