# Бухта

Telegram-бот для администратора, управляющий клиентами и их подписками.

## Назначение

Бот позволяет администратору вести базу клиентов и отслеживать статус их подписок прямо из Telegram — без стороннего интерфейса и сложной инфраструктуры.

## Реализованные функции

### Управление клиентами

| Функция | Статус |
|---|---|
| Список клиентов с инлайн-кнопками (⭐ для VIP) | ✅ |
| Карточка клиента: стоимость, статус подписки (активна / истекла / нет) | ✅ |
| Создание клиента (wizard: имя → стоимость) | ✅ |
| Переименование клиента (wizard, проверка уникальности) | ✅ |
| Изменение стоимости клиента (wizard) | ✅ |
| Удаление клиента с подтверждением (каскадное удаление подписок) | ✅ |
| VIP-статус: пометка клиента, исключение из уведомлений | ✅ |
| Раздел «Информация»: все клиенты со статусом, стоимостью и итоговой суммой | ✅ |

### Управление подписками

| Функция | Статус |
|---|---|
| Список подписок клиента (статус активна / истекла) | ✅ |
| Добавление подписки — пресеты (1 мес / 3 мес / 6 мес / 1 год) | ✅ |
| Добавление подписки — произвольная дата (ДД.ММ.ГГГГ, wizard) | ✅ |
| Продление подписки — пресеты (от даты окончания, платные периоды не теряются) | ✅ |
| Продление подписки — произвольная дата (wizard) | ✅ |
| Изменение даты окончания подписки (wizard) | ✅ |
| Удаление подписки с подтверждением | ✅ |
| Защита от дублирования: при активной подписке «Добавить» → продление | ✅ |

### Уведомления и расписание

| Функция | Статус |
|---|---|
| Ежедневные уведомления: подписки, истекающие сегодня и завтра | ✅ |
| Автопродление подписок VIP-клиентов при истечении | ✅ |
| VIP-клиенты исключаются из уведомлений об истечении | ✅ |
| Просмотр текущего расписания уведомлений (cron) | ✅ |
| Изменение расписания через cron-выражение с валидацией | ✅ |
| Сохранение расписания в БД (восстанавливается после рестарта) | ✅ |

### Инфраструктура

| Функция | Статус |
|---|---|
| Фильтрация по ADMIN_TELEGRAM_ID (чужие сообщения игнорируются) | ✅ |
| SQLite-база через Prisma (каскадное удаление) | ✅ |
| Docker + docker-compose | ✅ |
| CI/CD через GitHub Actions (деплой по пушу в main) | ✅ |
| Юнит-тесты (75 тестов, Vitest) | ✅ |

## Быстрый старт

```bash
git clone <repo-url>
cd buhta

cp .env.example .env
# Заполни .env: TELEGRAM_BOT_TOKEN, ADMIN_TELEGRAM_ID, DATABASE_URL

npm install
npx prisma migrate dev --name init
npm run start:dev
```

## Переменные окружения

| Переменная | Описание | Пример |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | Токен бота от @BotFather | `123456:ABC-DEF...` |
| `ADMIN_TELEGRAM_ID` | Telegram ID администратора | `123456789` |
| `DATABASE_URL` | Путь к SQLite-файлу | `file:./data/buhta.db` |

Получить свой Telegram ID можно у бота [@userinfobot](https://t.me/userinfobot).
