# Функции бота

## Команды

| Команда | Описание |
|---|---|
| `/start` | Приветствие и главное меню |
| `/menu` | Показать главное меню в любой момент |
| `/help` | Справка: команды, формат даты, флоу работы |
| `/cancel` | Отменить текущий ввод в любой wizard-сцене |

Все сообщения от не-администратора игнорируются молча (без ответа).

---

## Главное меню

После `/start` или `/menu` бот показывает три кнопки:

```
[ 👥 Клиенты   ]
[ 📊 Информация ]
[ ⚙️ Расписание ]
```

---

## Управление клиентами

### Список клиентов

**Вход:** кнопка «👥 Клиенты»  
**Отображение:** каждый клиент — отдельная кнопка. VIP-клиенты отображаются со звездой: `⭐ Иван Петров`.  
Внизу кнопки «➕ Добавить клиента» и «← Главное меню».

---

### Карточка клиента

```
👤 Иван Петров
Подписка: ✅ активна до 30.05.2026

[ 📋 Подписки ]  [ ✏️ Переименовать ]
[ ⭐ Сделать VIP ]
[ 🗑 Удалить ]   [ ← Назад ]
```

**Статусы подписки:**
- `✅ активна до ДД.ММ.ГГГГ` — есть активная подписка
- `❌ истекла ДД.ММ.ГГГГ` — подписка была, но закончилась
- `⬜ нет подписки` — подписок никогда не было

**VIP-кнопка** переключает статус:
- Нет VIP → `⭐ Сделать VIP`
- Уже VIP → `✖ Убрать VIP` (карточка показывает `👤 ⭐ Иван Петров`)

---

### Создать клиента

**Вход:** кнопка «➕ Добавить клиента»  
**Сценарий (Wizard):**
1. Бот: «Введите имя клиента:»
2. Администратор вводит имя → проверка уникальности
   - Успех → «Клиент создан.» → список клиентов
   - Имя занято → ошибка + повтор шага 2

---

### Переименовать клиента

**Вход:** кнопка «✏️ Переименовать» в карточке  
**Сценарий (Wizard):** аналогично созданию, но обновляет существующего клиента.

---

### Удалить клиента

**Вход:** кнопка «🗑 Удалить» в карточке  
**Сценарий:**
1. Бот: «Удалить "Иван Петров" и все его подписки?»  
   `[ ✅ Подтвердить ]  [ ❌ Отмена ]`
2. Подтверждение → каскадное удаление клиента и всех его подписок

---

## Информация о клиентах

**Вход:** кнопка «📊 Информация» в главном меню  
**Отображение:** все клиенты одним сообщением с иконкой статуса:

```
📊 Информация о клиентах:

✅ ⭐ Иван Петров — до 30.05.2026
❌ Мария Иванова — истекла 25.03.2026
⬜ Алексей Сидоров — нет подписки

Всего: 3
```

## Управление подписками

### Список подписок клиента

**Вход:** кнопка «📋 Подписки» в карточке клиента  
**Отображение:** каждая подписка — кнопка с датами и статусом.

```
📋 Подписки — Иван Петров:

[ 01.01.2026 — 31.03.2026 ❌ истекла ]
[ 01.04.2026 — 30.04.2026 ✅ активна ]

[ ➕ Добавить ]
[ ← Назад ]
```

---

### Меню подписки

Нажатие на подписку открывает её меню:

```
📋 01.04.2026 — 30.04.2026
✅ активна

[ 🔄 Продлить ]  [ ✏️ Изменить дату ]
[ 🗑 Удалить   ]
[ ← Назад ]
```

---

### Добавить подписку

**Вход:** кнопка «➕ Добавить» в списке подписок

**Если активной подписки нет:**
```
➕ Добавить подписку — Иван Петров

[ 1 месяц ]   [ 3 месяца ]
[ 6 месяцев ] [ 1 год    ]
[ 📅 Своя дата ]
[ ← Назад ]
```

**Если активная подписка уже есть** — бот автоматически переходит в режим продления:
```
🔄 Продление подписки — Иван Петров

⚠️ Активная подписка до 30.05.2026.
Продление начнётся с этой даты.

[ 1 месяц ]   [ 3 месяца ]
...
```
Это исключает потерю оплаченного времени.

---

### Продлить подписку

**Вход:** кнопка «🔄 Продлить» в меню подписки  
**Ключевое правило:** `newEndDate = существующий endDate + срок` — платные периоды не теряются.

---

### Изменить дату подписки

**Вход:** кнопка «✏️ Изменить дату» в меню подписки  
**Сценарий (Wizard):**
1. Бот: «Введите новую дату окончания (ДД.ММ.ГГГГ):»
2. Администратор вводит дату → валидация формата и будущей даты
3. Успех → дата обновлена → список подписок

Используется для ручной корректировки даты без пересчёта от текущей.

---

### Удалить подписку

**Вход:** кнопка «🗑 Удалить» в меню подписки  
**Сценарий:**
1. Бот: «Удалить подписку 01.04.2026 — 30.04.2026?»  
   `[ ✅ Подтвердить ]  [ ❌ Отмена ]`
2. Подтверждение → подписка удалена → список подписок

---

## Варианты срока подписки

| Кнопка | Поведение |
|---|---|
| 1 месяц | +1 календарный месяц от endDate |
| 3 месяца | +3 календарных месяца от endDate |
| 6 месяцев | +6 календарных месяцев от endDate |
| 1 год | +12 календарных месяцев от endDate |
| Своя дата | Ввод ДД.ММ.ГГГГ вручную |

## VIP-клиенты

VIP-статус предназначен для клиентов с автоматической оплатой.

**Что даёт VIP:**
- В списке клиентов отображается ⭐ перед именем
- В разделе «Информация» также показывается ⭐
- Клиент **исключается** из ежедневных уведомлений об истечении подписки

**Управление:** кнопка «⭐ Сделать VIP» / «✖ Убрать VIP» в карточке клиента.

## Расписание уведомлений

**Вход:** кнопка «⚙️ Расписание» в главном меню

**Экран расписания:**
```
⚙️ Расписание уведомлений

Текущее: 0 9 * * *

[ ✏️ Изменить ]  [ ← Назад ]
```

**Изменение расписания (Wizard):**
1. Бот показывает текущее выражение и подсказку по формату
2. Администратор вводит новое cron-выражение (5 полей: минута час день месяц день_недели)
3. Выражение валидируется — при ошибке бот просит ввести заново
4. Успех → расписание сохраняется в БД и применяется немедленно

**Примеры cron-выражений:**

| Выражение | Когда срабатывает |
|---|---|
| `0 9 * * *` | Ежедневно в 09:00 (по умолчанию) |
| `0 8 * * 1-5` | В 08:00 только по будням |
| `30 10 * * *` | Ежедневно в 10:30 |
| `0 9,18 * * *` | В 09:00 и 18:00 каждый день |

> Время — по часовому поясу сервера. Если сервер в UTC, а нужно московское время (UTC+3) — вычти 3 часа: `0 6 * * *` = 09:00 МСК.

Расписание сохраняется в базе данных и восстанавливается при перезапуске контейнера.

---

## Ежедневные уведомления

Бот сам присылает сообщение, если у кого-то из клиентов подписка истекает **сегодня** или **завтра**:

```
⚠️ Подписка истекает СЕГОДНЯ:
• Мария Иванова (до 26.04.2026)

🔔 Подписка истекает ЗАВТРА:
• Алексей Сидоров (до 27.04.2026)
```

VIP-клиенты в уведомления **не включаются**.

---

## Обработка ошибок

| Ситуация | Поведение бота |
|---|---|
| Сообщение от не-администратора | Игнорируется (нет ответа) |
| Некорректный формат даты | Повторный запрос с подсказкой формата |
| Дата в прошлом | Повторный запрос с объяснением |
| Дублирующееся имя клиента | Повторный запрос с указанием на дубль |
| Невалидное cron-выражение | Повторный запрос с примером |
| Пустой ввод | Повторный запрос |
