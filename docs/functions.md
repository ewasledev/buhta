# Функции бота

## Команды

| Команда | Описание |
|---|---|
| `/start` | Приветствие и главное меню |
| `/menu` | Показать главное меню в любой момент |
| `/help` | Справка по всем командам |

Все сообщения от не-администратора игнорируются молча (без ответа).

---

## Главное меню

После `/start` или `/menu` бот показывает инлайн-клавиатуру:

```
[ Клиенты ]
```

---

## Управление клиентами

### Список клиентов

**Вход:** кнопка «Клиенты» в главном меню  
**Отображение:** каждый клиент — отдельная кнопка с именем; внизу кнопка «+ Добавить клиента»  
**Действие:** нажатие на клиента открывает карточку клиента

---

### Карточка клиента

**Отображение:**
```
Иван Петров
Подписка: активна до 30.05.2026
────────────────────
[ Подписка ]  [ Переименовать ]
[ Удалить ]   [ ← Назад ]
```

Если подписки нет:
```
Иван Петров
Подписка: отсутствует
```

---

### Создать клиента

**Вход:** кнопка «+ Добавить клиента»  
**Сценарий (Wizard):**
1. Бот: «Введите имя клиента:»
2. Администратор вводит имя
3. Проверка уникальности
   - Успех → «Клиент "Иван Петров" создан.» → карточка клиента
   - Имя занято → «Клиент с таким именем уже существует. Введите другое:» → повтор шага 2

**Отмена:** кнопка «Отмена» на любом шаге возвращает к списку клиентов.

---

### Переименовать клиента

**Вход:** кнопка «Переименовать» в карточке клиента  
**Сценарий (Wizard):**
1. Бот: «Введите новое имя для "Иван Петров":»
2. Администратор вводит имя
3. Проверка уникальности
   - Успех → «Клиент переименован в "Иван Иванов".» → обновлённая карточка
   - Имя занято → ошибка + повтор

---

### Удалить клиента

**Вход:** кнопка «Удалить» в карточке клиента  
**Сценарий:**
1. Бот: «Удалить "Иван Петров" и все его подписки? Это действие нельзя отменить.»  
   `[ Подтвердить удаление ]  [ Отмена ]`
2. Подтверждение → клиент и все подписки удалены → список клиентов
3. Отмена → карточка клиента

---

## Управление подписками

### Список подписок клиента

**Вход:** кнопка «Подписка» в карточке клиента  
**Отображение:**
```
Подписки — Иван Петров:
• 01.01.2026 — 31.03.2026  (истекла)
• 01.04.2026 — 30.04.2026  (активна)
────────────────────
[ + Добавить ]  [ ← Назад ]
```

Каждая подписка — кнопка, открывающая меню подписки.

---

### Меню подписки

```
01.04.2026 — 30.04.2026 (активна)
────────────────────
[ Продлить ]  [ Удалить ]  [ ← Назад ]
```

---

### Добавить подписку

**Вход:** кнопка «+ Добавить» в списке подписок  
**Сценарий (Wizard):**
1. Бот: «Выберите срок подписки:»
   ```
   [ 1 месяц ]   [ 3 месяца ]
   [ 6 месяцев ] [ 1 год    ]
   [ Своя дата ] [ Отмена   ]
   ```
2. **Вариант A — preset выбран:**  
   `startDate = сегодня`, `endDate = startDate + выбранный срок`  
   → «Подписка добавлена: 25.04.2026 — 25.05.2026.» → список подписок

3. **Вариант B — «Своя дата»:**  
   Бот: «Введите дату окончания (ДД.ММ.ГГГГ):»  
   - Некорректный формат → «Неверный формат. Введите дату в формате ДД.ММ.ГГГГ:» → повтор  
   - Дата в прошлом → «Дата должна быть в будущем. Попробуйте ещё раз:» → повтор  
   - Успех → подписка создана → список подписок

**Если у клиента уже есть активная подписка:** бот предупреждает, но не блокирует создание.

---

### Продлить подписку

**Вход:** кнопка «Продлить» в меню подписки  
**Сценарий:** аналогично «Добавить», но:
- `newEndDate = существующий endDate + выбранный срок`
- При «Своей дате» новая `endDate` заменяет существующую

→ «Подписка продлена до 25.06.2026.» → меню подписки

---

### Удалить подписку

**Вход:** кнопка «Удалить» в меню подписки  
**Сценарий:**
1. Бот: «Удалить подписку 01.04.2026 — 30.04.2026?»  
   `[ Подтвердить ]  [ Отмена ]`
2. Подтверждение → подписка удалена → список подписок
3. Отмена → меню подписки

---

## Варианты срока подписки

| Кнопка | Поведение |
|---|---|
| 1 месяц | +1 календарный месяц |
| 3 месяца | +3 календарных месяца |
| 6 месяцев | +6 календарных месяцев |
| 1 год | +12 календарных месяцев |
| Своя дата | Ввод ДД.ММ.ГГГГ вручную |

---

## Обработка ошибок

| Ситуация | Поведение бота |
|---|---|
| Сообщение от не-администратора | Игнорируется (нет ответа) |
| Некорректный формат даты | Повторный запрос с подсказкой формата |
| Дата в прошлом | Повторный запрос с объяснением |
| Дублирующееся имя клиента | Повторный запрос с указанием на дубль |
| Пустое сообщение | Повторный запрос |
