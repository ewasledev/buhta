# Бухта

Telegram-бот для администратора, управляющий клиентами и их подписками.

## Назначение

Бот позволяет одному администратору вести базу клиентов и отслеживать статус их подписок прямо из Telegram — без стороннего интерфейса и сложной инфраструктуры.

## Реализованные функции

### Управление клиентами

| Функция | Статус |
|---|---|
| Список клиентов (инлайн-кнопки) | ✅ |
| Карточка клиента (имя + статус подписки) | ✅ |
| Создание клиента (wizard, проверка уникальности) | ✅ |
| Переименование клиента (wizard, проверка уникальности) | ✅ |
| Удаление клиента с подтверждением (каскадное удаление подписок) | ✅ |

### Управление подписками

| Функция | Статус |
|---|---|
| Список подписок клиента (статус активна/истекла) | ✅ |
| Добавление подписки — пресеты (1 мес / 3 мес / 6 мес / 1 год) | ✅ |
| Добавление подписки — произвольная дата (DD.MM.YYYY, wizard) | ✅ |
| Продление подписки — пресеты | ✅ |
| Продление подписки — произвольная дата (wizard) | ✅ |
| Удаление подписки с подтверждением | ✅ |

### Инфраструктура

| Функция | Статус |
|---|---|
| Фильтрация по ADMIN_TELEGRAM_ID (все чужие сообщения игнорируются) | ✅ |
| In-memory сессии для wizard-сцен | ✅ |
| SQLite-база через Prisma (каскадное удаление) | ✅ |
| Docker + docker-compose | ✅ |
| CI/CD через GitHub Actions (деплой по пушу в main) | ✅ |

## Быстрый старт

```bash
git clone <repo-url>
cd buhta

cp .env.example .env
# Заполни .env: TELEGRAM_BOT_TOKEN, ADMIN_TELEGRAM_ID, DATABASE_URL

npm install
npx prisma migrate dev --name init
npm run start:dev
```

## Переменные окружения

| Переменная | Описание | Пример |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | Токен бота от @BotFather | `123456:ABC-DEF...` |
| `ADMIN_TELEGRAM_ID` | Telegram ID администратора | `123456789` |
| `DATABASE_URL` | Путь к SQLite-файлу | `file:./data/buhta.db` |

Получить свой Telegram ID можно у бота [@userinfobot](https://t.me/userinfobot).
